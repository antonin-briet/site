Scolaire

Serveur Web Apache
sous Debian

Installation et configuration d'Apache2 sur Debian avec virtual hosts multiples et sécurisation HTTPS par certificat auto-signé OpenSSL.

DebianApache2HTTPSVirtual HostsOpenSSL

📋 Contexte & objectifs

Déploiement d'un serveur web fonctionnel sous Linux Debian avec hébergement de plusieurs sites via les virtual hosts Apache et sécurisation HTTPS.

Ce type de configuration est courant pour héberger des applications internes ou des sites intranet accessibles depuis le réseau local.

🛠️ Déroulement

  • 1

    Installation Debian

    Déploiement VM Debian sur Proxmox, configuration réseau, mise à jour du système.

  • 2

    Installation Apache2

    Installation du paquet, vérification du service, test de la page par défaut.

  • 3

    Virtual Hosts

    Création de deux virtual hosts distincts, configuration des fichiers de conf Apache, activation avec a2ensite.

  • 4

    HTTPS & certificat

    Génération d'un certificat auto-signé OpenSSL, activation du module SSL, redirection HTTP vers HTTPS.

  • 5

    Tests & logs

    Accès aux deux sites depuis un poste client, vérification du certificat, analyse des logs Apache.

⚠️ Difficultés rencontrées

  • Erreur 403 Forbidden — droits insuffisants sur le répertoire web (chmod et propriétaire www-data à corriger).
  • Alerte de sécurité navigateur sur le certificat auto-signé — différence avec une CA reconnue.
  • Résolution DNS des noms locaux — ajout manuel dans /etc/hosts des postes clients.

✅ Résultats obtenus

  • Deux sites web accessibles via noms de domaine distincts sur le même serveur.
  • Connexions sécurisées HTTPS opérationnelles.
  • Maîtrise de la gestion des services Linux et des fichiers de configuration Apache.

💡 Ce que j'ai appris

Ce TP m'a permis de maîtriser l'administration d'un serveur web Linux. Virtual hosts et HTTPS sont des compétences directement applicables en entreprise. J'ai mieux compris la chaîne de confiance SSL.
