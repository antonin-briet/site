Sécurité

Vaultwarden —
Gestionnaire de mots de passe

Déploiement d'une instance Bitwarden self-hosted sur serveur Linux, accessible uniquement via réseau VPN avec authentification 2FA et partage sécurisé.

DockerVaultwardenBitwardenHTTPS2FA TOTPArgon2

📋 Contexte & objectifs

Déploiement de Vaultwarden, une implémentation open-source légère compatible Bitwarden, pour héberger mes mots de passe sur mon propre serveur Linux et les synchroniser sur tous mes appareils sans dépendre d'un service tiers.

L'instance est accessible uniquement depuis un réseau VPN privé — aucun accès possible depuis internet, même en cas de compromission du serveur.

🛠️ Déroulement

  • 1

    Déploiement Docker

    Création du docker-compose, configuration des volumes et des variables d'environnement, premier démarrage.

  • 2

    Sécurisation du token admin

    Hash du token admin via Argon2, désactivation des inscriptions publiques.

  • 3

    Intégration HTTPS

    Configuration du reverse proxy Caddy avec certificat Let's Encrypt via challenge DNS Cloudflare.

  • 4

    2FA & utilisateurs

    Activation du 2FA TOTP, création des comptes, partage d'une collection en lecture seule pour un second utilisateur.

  • 5

    Migration des données

    Export depuis l'ancien gestionnaire, import CSV dans Vaultwarden, vérification de l'intégrité.

⚠️ Difficultés rencontrées

  • Certificats TLS initialement gérés via Tailscale, migrés vers Let's Encrypt via Caddy pour plus de robustesse.
  • Sécurisation du token admin — le hash Argon2 est obligatoire pour la production.
  • Restriction de création d'organisations aux seuls administrateurs — paramètre non documenté clairement.

✅ Résultats obtenus

  • Instance Vaultwarden opérationnelle en HTTPS avec certificat Let's Encrypt valide.
  • 2FA actif sur tous les comptes, token admin hashé Argon2.
  • Migration complète des mots de passe depuis l'ancien gestionnaire sans perte de données.

💡 Ce que j'ai appris

Ce projet m'a concrètement montré comment sécuriser un service sensible en couches : réseau isolé + HTTPS + 2FA + hash des secrets. La gestion des droits multi-utilisateurs m'a familiarisé avec les concepts de contrôle d'accès applicatif.
