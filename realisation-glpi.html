Installation &
Configuration de GLPI

Déploiement de GLPI sur un serveur Linux Debian avec configuration complète : inventaire du parc, helpdesk, gestion des tickets et intégration LDAP avec Active Directory.

GLPIDebianApache2MySQLLDAPPHP

📋 Contexte & objectifs

Déploiement de GLPI (Gestionnaire Libre de Parc Informatique) sur un serveur Linux Debian dans le cadre d'une activité E6. GLPI est un outil ITSM open-source largement utilisé en entreprise pour la gestion du parc informatique et le helpdesk.

L'objectif était de déployer une solution complète de gestion du parc, configurer le module helpdesk avec des files de tickets, et intégrer GLPI avec un annuaire Active Directory via LDAP.

🛠️ Déroulement

  • 1

    Installation du serveur LAMP

    Installation de Linux Debian, Apache2, MySQL/MariaDB et PHP avec les extensions nécessaires à GLPI.

  • 2

    Déploiement de GLPI

    Téléchargement et installation de GLPI, configuration de la base de données, assistant d'installation.

  • 3

    Paramétrage de l'inventaire

    Configuration des catégories d'équipements, création des types de matériels, import du parc.

  • 4

    Configuration du helpdesk

    Mise en place des files de tickets, catégories d'incidents, niveaux de priorité et SLA.

  • 5

    Intégration LDAP

    Configuration de l'annuaire Active Directory comme source d'authentification via LDAP, import des utilisateurs.

  • 6

    Tests & validation

    Création de tickets de test, vérification des droits par profil, test de la connexion via comptes AD.

⚠️ Difficultés rencontrées

  • Configuration LDAP — correspondance des attributs AD avec les champs GLPI (DN, filtre de connexion).
  • Droits insuffisants sur les répertoires GLPI — erreurs 403 corrigées via chmod et www-data.
  • Gestion des dépendances PHP — certaines extensions manquantes détectées lors de l'installation.

✅ Résultats obtenus

  • Instance GLPI opérationnelle accessible via navigateur sur le réseau local.
  • Inventaire du parc configuré avec les catégories d'équipements adaptées.
  • Module helpdesk fonctionnel avec files de tickets et niveaux de priorité.
  • Authentification LDAP avec Active Directory opérationnelle — les utilisateurs se connectent avec leurs identifiants de domaine.

💡 Ce que j'ai appris

Ce projet m'a permis de comprendre concrètement le fonctionnement d'un outil ITSM en entreprise. GLPI est directement lié à mon expérience chez V2SI où je l'utilise quotidiennement pour la gestion des tickets. L'intégration LDAP m'a permis de consolider mes connaissances en Active Directory dans un contexte applicatif.
