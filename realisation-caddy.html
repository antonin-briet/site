Reverse Proxy Caddy
+ DNS Cloudflare

Reverse proxy HTTPS avec certificats Let's Encrypt automatiques via challenge DNS Cloudflare depuis une IP privée. Accès unifié aux services par sous-domaines.

CaddyLet's EncryptCloudflare APIHTTPSDocker

📋 Contexte & objectifs

Caddy est le reverse proxy qui unifie l'accès à tous mes services sous un domaine personnalisé (votredomaine.dev). Il gère automatiquement les certificats TLS via Let's Encrypt — ce qu'on appelle le TLS automatique.

La difficulté principale : le serveur est derrière un VPN sans IP publique exposée. Le challenge HTTP classique de Let's Encrypt est impossible — j'ai utilisé le challenge DNS via l'API Cloudflare.

🛠️ Déroulement

  • 1

    Image Caddy personnalisée

    Build d'une image Caddy avec le plugin cloudflaredns pour le challenge DNS.

  • 2

    Configuration Caddyfile

    Déclaration des virtual hosts, configuration des proxys vers chaque service interne.

  • 3

    Token API Cloudflare

    Création d'un token API Cloudflare avec droits DNS zone:edit, injection via variable d'environnement.

  • 4

    Certificats Let's Encrypt

    Validation du challenge DNS, obtention et renouvellement automatique des certificats.

  • 5

    Sécurisation UFW

    Configuration UFW pour bloquer les ports 80/443 depuis internet, autoriser uniquement l'interface VPN.

⚠️ Difficultés rencontrées

  • Let's Encrypt refusait les challenges HTTP — l'IP n'était pas routable publiquement.
  • Caddy ne pouvait pas résoudre les DNS ACME à cause du conflit avec AdGuard.
  • Exposition involontaire des ports 80/443 avant la fermeture des règles UFW.

✅ Résultats obtenus

  • Tous les services accessibles via HTTPS avec certificat Let's Encrypt valide.
  • Renouvellement automatique des certificats sans intervention manuelle.
  • Ports 80/443 inaccessibles depuis internet — uniquement via VPN.

💡 Ce que j'ai appris

La mise en place du challenge DNS m'a fait comprendre en profondeur le protocole ACME et les alternatives au challenge HTTP. J'ai mieux saisi la notion de HSTS et pourquoi certains domaines imposent HTTPS de façon permanente. Compétence directement applicable en entreprise.
