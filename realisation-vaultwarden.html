Vaultwarden —
Gestionnaire de mots de passe

Déploiement d'une instance Bitwarden self-hosted sur serveur Linux, accessible uniquement via réseau VPN avec authentification 2FA et partage sécurisé.

DockerVaultwardenBitwardenHTTPS2FA TOTPArgon2

📋 Contexte & objectifs

Déploiement de Vaultwarden, une implémentation open-source compatible Bitwarden, pour héberger des mots de passe sur un serveur Linux personnel et les synchroniser sur tous les appareils sans dépendre d'un service tiers.

L'instance est accessible uniquement via un réseau VPN privé — aucun accès possible depuis internet.

🛠️ Déroulement

  • 1

    Déploiement Docker

    Création du docker-compose, configuration des volumes et variables d'environnement, premier démarrage.

  • 2

    Sécurisation du token admin

    Hash du token via Argon2, désactivation des inscriptions publiques.

  • 3

    Intégration HTTPS

    Configuration du reverse proxy Caddy avec certificat Let's Encrypt via challenge DNS Cloudflare.

  • 4

    2FA & utilisateurs

    Activation du 2FA TOTP, création des comptes, partage d'une collection en lecture seule.

  • 5

    Migration des données

    Export depuis l'ancien gestionnaire, import CSV, vérification de l'intégrité.

⚠️ Difficultés rencontrées

  • Certificats TLS initialement gérés via Tailscale, migrés vers Let's Encrypt via Caddy.
  • Hash Argon2 du token admin obligatoire pour la production — non documenté clairement.
  • Restriction de création d'organisations aux seuls administrateurs.

✅ Résultats obtenus

  • Instance opérationnelle en HTTPS avec certificat Let's Encrypt valide.
  • 2FA actif sur tous les comptes, token admin sécurisé.
  • Migration complète des mots de passe sans perte de données.

💡 Ce que j'ai appris

Ce projet m'a concrètement montré comment sécuriser un service sensible en couches : réseau isolé + HTTPS + 2FA + hash des secrets. La gestion des droits multi-utilisateurs m'a familiarisé avec les concepts de contrôle d'accès applicatif.
