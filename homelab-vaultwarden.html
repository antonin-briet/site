🔐 Vaultwarden

Gestionnaire de mots de passe self-hosted compatible Bitwarden, accessible uniquement via Tailscale avec authentification 2FA et partage de collection en lecture seule.

DockerVaultwardenBitwardenHTTPS2FA TOTPArgon2

📋 Contexte & objectifs

Dans le cadre de mon homelab, j'ai déployé Vaultwarden, une implémentation open-source légère et compatible Bitwarden, pour remplacer un gestionnaire de mots de passe propriétaire. L'objectif était d'héberger moi-même mes mots de passe et de les synchroniser sur tous mes appareils sans dépendre d'un service tiers.

L'instance est accessible uniquement depuis le réseau Tailscale, ce qui garantit qu'aucun accès n'est possible depuis internet, même si le serveur est compromise.

⚠️ Contraintes

  • Accès strictement limité au réseau VPN Tailscale
  • HTTPS obligatoire (imposé par le domaine .dev)
  • Gestion de plusieurs utilisateurs avec droits distincts
  • Migration depuis un gestionnaire existant via export/import CSV

🔧 Difficultés rencontrées

  • Certificats TLS initialement gérés via Tailscale, migrés vers Let's Encrypt via Caddy
  • Sécurisation du token admin — passage en hash Argon2 obligatoire
  • Restriction de création d'organisations aux seuls administrateurs

✅ Solutions apportées

Reverse proxy Caddy avec DNS challenge Cloudflare pour Let's Encrypt. Token admin hashé via Argon2, signups désactivés, 2FA via application TOTP. Organisation partagée avec collection en lecture seule pour l'utilisateur secondaire.

💡 Ce que j'ai appris

Ce projet m'a concrètement montré comment sécuriser un service sensible en couches : réseau isolé + HTTPS + 2FA + hash des secrets. La gestion des droits multi-utilisateurs dans Vaultwarden m'a également familiarisé avec les concepts de contrôle d'accès applicatif.
